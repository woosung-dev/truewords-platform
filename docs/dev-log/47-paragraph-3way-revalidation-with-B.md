# 47. 3-way A/B/F 재검증 — 운영 권장 진화 (F 단일 → F+B 하이브리드 후보)

- 일자: 2026-04-30

- 영역: Phase 2.2 — 청킹 전략 재평가

## Context

dev-log 45에서 옵션 F(paragraph) 운영 전환을 결정했지만 **B(prefix) 측정이 새 평가셋에서 누락**됐다. 이전 100선에서 B가 측정됐고 A/F와 거의 동등(0.819/0.821/0.847)이었으나, 새 평가셋(50문항, 캐시 격리)에서는 B vs F 직접 비교 데이터가 없었다.

또한 다음 질문이 남아 있었다:
- **데이터 14% 적재(88/615권) 상태**에서 단정 가능한가?
- B의 잠재력 미검증 상태에서 F 단일 채택이 적절한가?

본 작업은 동일 측정 방식(50문항, 캐시 격리, F→A→B 순서)으로 B를 추가 측정하여 3-way 비교를 완성한다.

> **운영 적용 후보는 F(paragraph-based)가 가장 낫다.**

> **권장 운영안**: F 기본 + B 보조 검색 후보 병합 + exact lexical fallback. 단일 옵션만 고르면 F.

## 결정

### 운영 전환 — dev-log 45의 F 채택 유지

`'all'` 봇 `collection_main = malssum_poc_v3` 그대로. 이미 영구 적용됨(2026-04-30 dev-log 45).

### 운영 권장 진화 — F 단일에서 **F 기본 + B 보조** 후보로

dev-log 45는 "F 단일 운영"을 결정했으나, 본 3-way 측정에서 **B의 L1/L2 강점이 F의 약점을 정확히 보완**한다는 패턴이 확인됐다. 따라서 **L2 약점 보강 plan(dev-log 46)에 다음 옵션 추가**:

- **방안 A** (재임베딩, 메타데이터 prefix injection) — 기존 dev-log 46
- **방안 B** (질문 파싱 + volume 필터) — 기존 dev-log 46
- **방안 D (신규)** — **F + B 하이브리드 검색** — paragraph(F)와 prefix(B) 컬렉션 모두에서 검색 후 RRF/Re-ranker로 병합. L1/L2는 B 결과 가중, L3~L5는 F 결과 가중

방안 D가 매력적인 이유:
- **재임베딩 불필요** — B 컬렉션(malssum_poc_v2) 이미 존재
- **L1/L2 약점은 B의 강점**으로 직접 보완
- **Codex 권고와 일치** ("F 기본 + B 보조 검색 후보 병합")

단점:
- 검색 비용 2배 (두 컬렉션 모두 query)
- RRF 가중치 튜닝 필요

### 데이터 적재 14% 한계 명시

본 측정의 절대 신뢰도는 **현재 88권(약 14%) 적재 상태 한정**이다. Codex도 "전 데이터 적재 후에도 상대 결론(A 부적합, F/B 우월)은 유지될 가능성"으로만 평가. 다음 마일스톤에서 재측정 필수:

- **데이터 50% 적재 (~308권) 후**: 동일 50문항 재측정 → F/B 격차 변화 확인
- **데이터 75% 적재 (~461권) 후**: B의 L2 강점이 유지되는지 확인
- **데이터 100% 적재 (615권) 후**: 최종 결정

각 마일스톤은 schedule routine으로 자동 트리거.

### 100선 vs 50선 격차 변화

| 측정 | 표본 | A | B | F | F-B 격차 |
|---|---:|---:|---:|---:|---:|
| 100선 (4/29) | 100 | 0.819 | 0.821 | 0.847 | +0.026 |
| **50선 (4/30)** | 50 | 0.685 | 0.734 | 0.742 | **+0.008** |

**격차가 1/3로 축소**. F-B는 사실상 동등 수준.
