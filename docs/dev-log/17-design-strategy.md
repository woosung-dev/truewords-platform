# UI/UX 디자인 전략

## 1. 디자인 비전

**"검증된 말씀으로 마음의 평안을, 함께하는 묵상으로 영적 성장을"**

말씀 앱의 디자인은 **신뢰감(Trust)**, **평안함(Peace)**, **친근함(Warmth)** 세 가지 감성을 전달해야 한다. 화려한 시각 자극보다는 절제된 아름다움으로, 사용자가 말씀에 집중할 수 있는 환경을 만든다.

---

## 2. 디자인 원칙

### 2.1 5대 원칙

| # | 원칙 | 설명 | 반면교사 |
|---|------|------|---------|
| 1 | **말씀이 주인공** | UI는 배경, 성경 말씀이 전면에 | 화려한 UI가 콘텐츠를 압도 |
| 2 | **전 연령 포용** | 1020 청년 ~ 5060 장년까지 접근 가능 | 특정 세대만 타겟 (초원: 1030) |
| 3 | **신뢰를 시각화** | 출처 표시, 검증 배지, 투명한 AI 한계 고지 | AI 답변을 절대 진리처럼 제시 |
| 4 | **습관을 디자인** | 자연스러운 일일 루틴 흐름 | 기능 나열식 메뉴 |
| 5 | **공동체를 연결** | 함께 묵상하는 경험의 시각화 | 고립된 개인 경험 |

### 2.2 경쟁사 디자인 벤치마킹

```
디자인 참조 매트릭스:

Hallow에서 배울 점:
├── 미니멀한 다크 톤 (집중/명상 유도)
├── 오디오 플레이어 UX (Spotify급)
├── 시간대별 큐레이션 (아침/저녁)
└── Streak/진도 시각화

초원에서 배울 점:
├── 채팅 UI의 자연스러운 톤
├── 새싹 키우기 (성장 메타포)
├── 공유 카드 디자인
└── 성경 뷰어 가독성

Calm/Headspace에서 배울 점:
├── 평온한 컬러 팔레트
├── 부드러운 트랜지션 애니메이션
├── 수면 모드 UI (어두운 톤)
└── 호흡/카운트 인터랙션

차별화 포인트:
├── 한국 정서에 맞는 따뜻한 톤 (차가운 미니멀 X)
├── 교회 공동체 연결감 시각화
├── 전 연령대 접근성 (큰 글씨, 직관적 네비게이션)
└── AI 출처/신뢰도 시각화 UI
```

## 3. 컬러 시스템

### 3.1 기본 팔레트

| 역할 | 컬러 | 용도 |

| **Text Secondary** | 웜 그레이 (#6B6B6B) | 보조 텍스트, 캡션 |
| **Success** | 올리브 그린 (#5B8C5A) | Streak, 완료, 성장 |

### 3.3 감성 의도

```
따뜻한 아이보리 배경 → "오래된 성경책을 펼치는 느낌"

차분한 남색 주요색 → "깊은 묵상, 평안한 밤하늘"
```

## 6. 인터랙션 & 애니메이션 가이드

### 6.1 핵심 애니메이션

| 장면 | 애니메이션 | 기술 | 목적 |
|------|----------|------|------|
| AI 답변 수신 | 타이핑 효과 (글자 단위) | SSE + 커스텀 렌더링 | 실시간 대화감 |
| 묵상 완료 | 나무 성장 + 빛 효과 | Lottie | 성취감 + 보상 |
| Streak 마일스톤 | 축하 파티클 | Lottie | 동기 부여 |
| 카드 전환 | 스와이프 + 페이드 | 네이티브 Gesture | 부드러운 탐색 |
| 로딩 | Skeleton UI + 펄스 | CSS/네이티브 | 대기 불안 해소 |
| 에러 | 부드러운 쉐이크 + 안내 | 네이티브 | 부드러운 에러 고지 |

### 6.2 트랜지션 원칙

- **지속시간**: 200~350ms (체감 즉각성 + 부드러움 균형)
- **이징**: ease-out (빠르게 시작, 부드럽게 종료)
- **방향성**: 깊이 들어감 = 오른쪽에서 슬라이드, 돌아감 = 왼쪽으로 슬라이드
- **피드백**: 모든 탭 가능 요소에 터치 피드백 (100ms 내)

## 7. 에러 & 엣지 케이스 UX

### 7.1 AI 관련 에러

| 상황 | UX 대응 |
|------|---------|
| AI 응답 지연 (>5초) | "말씀을 깊이 찾고 있습니다..." + 프로그레스 |
| AI 타임아웃 | "잠시 후 다시 시도해 주세요" + 재시도 버튼 |
| 네트워크 에러 | 오프라인 모드 안내 + 캐시된 콘텐츠 제공 |
| 부적절한 질문 | "이 질문에는 답변드리기 어렵습니다" + 가이드 |
| 자해/위기 감지 | 즉시 전문 상담 연결 안내 (doc 09) |

### 7.2 일반 에러

| 상황 | UX 대응 |
|------|---------|
| 콘텐츠 로딩 실패 | 캐시된 전일 묵상 카드 Fallback |
| 이미지 공유 실패 | 텍스트 복사 대안 제공 |
| Streak 데이터 동기화 실패 | 로컬 우선 저장 + 백그라운드 동기화 |

## 8. 프론트엔드 기술 결정

### 8.1 프레임워크 비교

| 항목 | React Native | Flutter |
|------|-------------|---------|
| 성숙도 | 높음 | 높음 |
| 오디오 처리 | 서드파티 의존 | 서드파티 의존 |
| 애니메이션 | Reanimated + Lottie | Rive + 내장 |
| 생태계 | 넓음 (npm) | 성장 중 (pub.dev) |
| 네이티브 브릿지 | JS ↔ Native | Dart ↔ Native |
| SSE 지원 | fetch + EventSource | dio + SSE |

→ **두 프레임워크 모두 적합**, 팀 역량 기준으로 결정

### 8.2 핵심 기술 요구사항

| 기능 | 필수 기술 |
|------|----------|
| AI 스트리밍 | SSE (Server-Sent Events) 클라이언트 |
| 오디오 재생 | HLS 스트리밍 + 백그라운드 재생 네이티브 모듈 |
| 성경 뷰어 | Virtualized List (대용량 텍스트) |
| 이미지 공유 | html2canvas / Skia 렌더링 |
| 애니메이션 | Lottie / Rive 런타임 |
| 오프라인 | AsyncStorage / Hive + SQLite |
| 푸시 | FCM + APNs 네이티브 연동 |
| 상태 관리 | React Query / Riverpod (프레임워크별) |
