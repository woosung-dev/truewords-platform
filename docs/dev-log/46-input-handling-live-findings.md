# ADR-46 한국 종교 AI 챗봇 4종 실측 + TrueWords 적용 제안 (2026-04-28)

> ADR-45 부록 9.1의 미달성 항목(시드 질문 직접 시도)을 보강한다. Playwright MCP로 4종 챗봇을 실측해 입력 UX 패턴을 추출하고, TrueWords에 즉시 적용 가능한 구체 액션을 도출했다.

## 1. Executive Summary

- **5개 시드 질문 중 4개를 말씀 선집 챗봇(`truefather.web.app`)에서 직접 시도 완료** — ADR-45 부록 9.1 미달성 해소.
- **[2026-04-28 추가] 초원AI 모바일 앱 정보 통합** — 사용자 제공 스크린샷 3장 + 공유 답변 URL 2개로 마케팅 페이지에서 못 본 핵심 기능 다수 확보. **초원AI는 본 벤치마크에서 가장 정교한 입력 UX·답변 구조를 가진 챗봇으로 재평가**.
- **즉시 차용할 P0 6건 (2건 신규)** — (NEW) **답변 모드 5종 페르소나 토글** (위급 라우팅 자연 구현 경로!), (NEW) **하단 floating action bar (새 질문/북마크/공유)**, (1) 자동 follow-up 추천 3개, (2) 원문보기 모달, (3) placeholder 두 줄 가이드, (4) 정교한 면책 4문장.
- **차별화 기회 재정의** — (a) **위급 라우팅** 은 초원의 "목회 상담자" 페르소나 패턴으로 *부분 구현 확인* → 별도 crisis intent 신설 대신 페르소나 라우팅으로 재설계 권장. (b) **능동 명확화** 와 (c) **RAGAS 자동 평가** 는 여전히 4종 모두 부재 — TrueWords 단독 우위 유지.
- **자기모순 해소** — ADR-45 P0-2 (multi-turn 결합)이 cogScript 실측에서 정상 작동 확인. "그럼 그것은 왜 중요한가요?" → 직전 답변의 "이 나무" 자연 이어받기 ✓.
- **신규 액션 14건** — P0-E (페르소나 5종 ★★), P0-G (floating bar), P1-G (강조점 토글), P1-H (답변 3-탭), P1-J (기도문), P1-K (검수+이단학습 차별화), P1-L (본문 jump 카드), P1-M (본문→AI 해설요약 양방향), P2-D (공개/비공개), P2-E (공유 추적), P2-F (일러스트), P2-G (오디오 TTS), P2-H (통독표), P2-I (번역본 토글).

핵심 UX 패턴:
- **5개 언어 토글** (한/일/영/번체/간체) — 카드 형 플래그 버튼
- **답변 구조 = 도입(1줄) + 3개 번호 굵은 소제목 + "📖 관련 말씀 인용" 박스 + 💡 추천 follow-up 3개**
- **`.rec-link` SPAN 클릭 가능 칩** — 답변마다 자동 생성 (ADR-45는 0건 fallback 시에만 추천)
- **`.view-original-btn` "원문보기"** → 모달에 원문 전체 표시, 인용된 부분 **노란색 배경 하이라이트**
- 안내 카드 (5가지 사용 시나리오) — 첫 진입 시 노출
- 면책 ("AI 답변은 참고용이며, 정확한 내용은 원문을 확인하시기 바랍니다.") — 입력창 하단 상시 노출
- 모델 버전 명시 (`v1.3.4_2026-03-22 (gemini-3.1-flash-lite)`) — 안내 카드 푸터

#### 2.3.1 모바일 앱 "질문하기" 화면 (이미지 #1)

```

│                                            │
│                                            │
└──────────────────────────────────────────┘

맞춤 설정
  교단        전체 교단        >
  답변 모드   표준              >
  공개 여부   비공개            >
```

핵심 발견:
- **placeholder 자체가 사용 팁** — 한 줄이 아니라 두 줄: 첫 줄은 무엇을 입력할지("고민이나 질문"), 둘째 줄은 어떻게 입력할지("구체적일수록 정확")
- **3개 맞춤 설정 메뉴**: 교단 / 답변 모드 / 공개 여부
- **정교한 면책** — 4가지 차별화 메시지를 한 단락에 압축

#### 2.3.2 답변 모드 페르소나 5종 (이미지 #2) — **가장 큰 발견**

`답변 모드 설정 — 누가 답해주면 좋을까요?`

| 모드 | 설명 |
|------|------|
| **표준 (추천)** ✓ | 질문에 가장 적합한 방식으로 자동 선택 |
| **신학과 교수 (고급)** | 깊이 있는 신학적 해설 |
| **목회 상담자** | 위로와 상담 중심 |
| **초신자 양육자** | 신앙의 기초를 쉽고 친절하게 |
| **유년부 선생님** | 어린이 눈높이 설명 |

**TrueWords에 직접 적용 가능한 핵심 통찰**: ADR-45 P0-1 "위급 라우팅 (자해/위기)"의 구현 경로. "오늘 너무 힘듭니다" 같은 정서 시드 → IntentClassifier가 "목회 상담자" 모드 자동 선택 → 시스템 프롬프트가 위로·상담 톤 + 1393 핫라인 통합. 별도 crisis intent 신설 없이 **페르소나 라우팅으로 자연 통합** 가능.

#### P1-D · 답변 이미지 카드 SNS 공유

**근거**: 초원 "관련 성경구절은 이미지로 저장해 공유". 종교 콘텐츠 바이럴 채널 = 지인 권유.

#### P1-E · Prompt-injection 거절 → 정상 흐름 복귀

**근거**: 말씀 선집 챗봇은 인젝션 시 거절 후 `💡 다음 질문을 추천` 으로 부드럽게 복귀. TrueWords 47-패턴은 단순 차단.

**예상 공수**: 0.5주. **영향**: 인젝션 시도 사용자도 정상 흐름 유지 (악의 의도 X 사용자도 다수).

#### P1-K · "검수 사이클 + 이단/오류 데이터 학습" 차별화 메시지 (NEW · 2026-04-28)

**근거**: 초원AI 면책 마지막 두 문장 — "목회자 검수위원회를 통해 신학적 검증 완료, 국내 주요 이단 데이터 학습으로 신학적 방화벽 구축". 이건 **운영 프로세스 + 학습 데이터 = 마케팅 차별화** 트리오.

**TrueWords 매핑**:
1. **검수 사이클 운영** — 매주 1회, 무작위 답변 50건 운영자 검수 + 라벨링 (P1-A 사용자 피드백 + 운영자 라벨 결합)
2. **오류/이단 답변 학습 데이터화** — 검수에서 *부적합* 라벨 받은 답변을 negative example로 시스템 프롬프트 few-shot 또는 critic LLM에 주입
3. **About 페이지 / 면책 단락에 명시** — "TrueWords는 운영자 검수를 거친 답변만을 표준 답변으로 노출하며, 부적합 답변은 즉시 학습 데이터화하여 재발을 방지합니다"

**코드 변경 위치**:
- `backend/src/admin/routers/answer_review.py` (신규) — 운영자 검수 큐 + 라벨링 엔드포인트
- `chat_message_feedback` 테이블에 `reviewer_user_id, reviewer_label ENUM(approved, theological_error, citation_error, tone_error, off_domain)` 추가
- 운영자 라벨이 *theological_error* 인 답변을 시스템 프롬프트 negative few-shot에 자동 주입 (주 1회 운영)
- 면책 P0-D에 차별화 메시지 통합

**예상 공수**: 2주 (검수 큐 + 라벨링 UI + 운영 자동화). **영향**: 차별화 마케팅 메시지 + 답변 품질 시간이 갈수록 우상향.

### 4.3 P2 — 백로그

- **P2-K (NEW)** 추천 follow-up 부분 블러 + 로그인 유도 — 초원이 follow-up 5개 중 3·4·5번을 블러 + "소중한 말씀을 놓치지 마세요 / 카카오 로그인" 버튼. 비로그인 사용자 → 로그인 전환 깔때기. P0-A 자동 follow-up과 결합 시 위력
