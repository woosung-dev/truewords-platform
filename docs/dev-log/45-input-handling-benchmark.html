1. Executive Summary

가설은 부분적으로만 옳다. "사용자 입력만 잘 받으면 답변 만족도↑" 는 산업 연구로 뒷받침되지만(Query Rewriting 단독 +14.45%, Multi-Query 복잡 multi-hop +7%, Contextual Retrieval+Rerank 검색 실패율 -67%), 단일 차원 개선이 아니라 다층 입력-처리 스택(UX·이해·재작성·다중턴·명확화·가드레일·인용)이 누적되어야 효과가 난다.

TrueWords 가장 큰 격차 3가지

Critical 1
Multi-turn 부재
현재 각 질문 독립 처리. session.py 는 DB 기록만.
Critical 2
능동 명확화 없음
결과 0건일 때만 추천 3개. 모호 질문 사전 게이트 부재.
Critical 3
입력 UX 빈곤
자동완성·추천칩·음성·카테고리 필터 모두 없음.

즉시 가져갈 자산

  • Anthropic Contextual Retrieval — 검색 실패율 67% 감소 (+ Rerank)
  • Multi-Query + HyDE 적응형 라우팅 — Intent별 분기
  • INTENT-SIM 기반 명확화 트리거 (NAACL 2025)
  • Polaris Bible형 4-layer 가드레일 + Magisterium 데이터 비학습 정책
3-모델 크로스 리뷰 종합 (8절) · 종합 6/10, 가설 5/10. 핵심 합의: (i) 실측 베이스라인·한국어 종교 골드셋·RAGAS baseline 부재가 가장 큰 약점, (ii) P0-2 Multi-turn 은 *결합* 대신 *self-contained rewrite* 로 축소, (iii) 비용·지연·한국 법령 근거 보강 필요. 수정된 P0: P0-0 평가셋 신규 + P0-1 위기 라우팅(input_validator 레이어) + P0-2 single-turn rewrite + P0-4 Contextual Retrieval 승격.

5. 산업 베스트 프랙티스 7토픽

5.1 Query Rewriting / Reformulation

  • Rewrite-Retrieve-Read · 원 질문 LLM 재작성 → 검색
  • Step-Back Prompting (DeepMind, LangChain 구현) · 추상화 질문을 별도로 만들어 두 질문 모두 검색
  • HyDE · 가상 답변을 먼저 생성·임베딩 후 검색. 짧은 질의·어휘 미스매치에 효과
  • Multi-Query / RAG-Fusion · 다중 의도 분해, +14.45% / 복잡 multi-hop +7%
  • Adaptive routing · 짧은 쿼리 → HyDE / 모호 → Multi-Query / 일반 → Expansion

5.2 Multi-turn / Conversational RAG

  • Follow-up rewriting · 후속 질문을 self-contained query로 변환
  • mtRAG benchmark (TACL 2025) · 110 대화 · 평균 7.7턴 · 4 도메인
  • PIR (Passage-Informed Rewriting) · 1차 검색 결과를 LLM에 다시 주입 (closed loop)
  • ConvSelect-RAG · 3-stage = 대화 결합 + 메타 사전필터 + 청크 검색. latency -23.5%, accuracy +18.7%
  • History compression (Microsoft) · 오래된 대화는 요약, 새 턴은 풀 컨텍스트
경고 (arXiv 2505.06120) · 모든 LLM이 multi-turn에서 평균 39% 성능 하락. 결합 대신 single-turn-rewrite 후 처리가 안전.

5.3 Clarification & Disambiguation

  • INTENT-SIM (NAACL 2025) · 명확화 질문과 자가 시뮬레이션 답변의 엔트로피로 명확화 트리거 결정
  • AmbigChat (UIST 2025) · 계층적 명확화 widget — 질문 + 답변 widget 분리
  • Tree of Clarifications · 재귀적 분해
  • Tri-agent ambiguity detection · 0.92 정확도
놀라운 발견 · Q&A 명확화 루프가 "처음부터 완벽한 질문" 보다 사용자 만족도 더 높음 (NAACL 2025).

5.4 Input UX 패턴

  • Suggested starter prompts + 칩/카드/버튼 (NN/g)
  • Sample Q&A flow: "What does grace mean?" → 정의 + 2 인용 + 1줄 맥락 + 묵상 프롬프트 (반복 학습 시퀀스)
  • Faith tradition 토글 / 인물 카드 (Saint Chat) / Bible 자동 버전 선택

5.5 Guardrails / 안전

  • OWASP LLM Top 10 2025 · LLM01 Prompt Injection · LLM07 System Prompt Leakage · LLM08 Vector/Embedding Weaknesses · 53% 기업 RAG 사용
  • 다층 방어: 입력 검증 + 출력 필터 + 인간 oversight + 적대적 테스트
  • 권장 도구: LLM Guard, Guardrails AI
  • 자해/위급: Stanford 2025 — 챗봇은 적절 응답 불가 / CA SB 243 (2025-10) — AI 공개 강제 + 자해 방지 프로토콜 의무화
  • 종교 특화: Polaris 4-layer · Magisterium 데이터 비학습 · QuranGPT 도메인 잠금

5.6 Anthropic Contextual Retrieval

핵심 수치 · 청크에 50–100 토큰 컨텍스트 prepending → 임베딩+BM25
  • Contextual Embeddings 단독 -35% 검색 실패율
  • + Contextual BM25 -49%
  • + Reranking -67% ⭐
  • Prompt caching $1.02 / 1M tokens
TrueWords는 Gemini 사용 → 동등 캐싱 정책 검증 필요 (Gemini context caching API)

5.7 평가 지표 (RAGAS)

  • Faithfulness (사실성) · 응답이 검색 컨텍스트와 일관되는가
  • Answer Relevancy · 응답이 원 질의에 부합하는가 (사실성 X)
  • Context Precision / Recall — 권장 0.8↑
  • TrueWords는 PR #68에서 도입 완료 → 다음은 멀티턴 mtRAG 벤치 도입 검토

6. 가설 검증

가설 — "사용자 입력만 잘 받으면 답변 만족도가 크게 좋아진다."

지지 근거 5건

  1. Anthropic Contextual Retrieval — 입력 단의 청크 컨텍스트 보강만으로 검색 실패율 35–67% 감소
  2. Multi-Query+HyDE — +14.45%, 복잡 multi-hop +7%
  3. NAACL 2025 INTENT-SIM — 명확화 Q&A 루프가 완벽한 단일 질문보다 만족도 ↑
  4. ConvSelect-RAG — 멀티턴 결합으로 latency -23.5%, accuracy +18.7%
  5. NN/g UX 연구 — vague prompt = 낮은 만족도, suggested starter prompts → 참여도 ↑

  1. 베이스라인·평가셋 부재가 가장 큰 약점. 산업 수치(67%, 14.45%, -23.5% latency, +18.7% accuracy)를 TrueWords 도메인 검증 없이 차용.
