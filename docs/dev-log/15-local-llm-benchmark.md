# 로컬 LLM 벤치마크 보고서

## 1. 벤치마크 개요

Apple Silicon (M5 Pro 64GB) 환경에서 실행 가능한 로컬 LLM 13종의 성능을 종합 측정하여, 개발/테스트 환경에서의 모델 선택 가이드를 제공한다.

| 항목 | 값 |
|------|-----|
| 하드웨어 | Apple M5 Pro, RAM 64GB (통합 메모리) |
| MLX 버전 | 0.31.1, Python 3.14 |
| 테스트 규모 | 13개 모델 × 25문제 × 8가지 신기술 |
| 테스트 일자 | 2026-03-22 |
| 평가 영역 | 논리추론, 코딩, 수학, 한국어, 창의, 시스템설계, 교차도메인 |

## 3. 신기술 테스트 결과 요약

### 3.1 효과 종합

| 기술 | 효과 | 최적 대상 | 권장 |
|------|------|----------|------|
| **Speculative Decoding (0.8B)** | +57~62% (27B) | 27B mlx_lm | **강력 권장** |
| **Spec Decoding (4B JANG)** | +21% (27B-JANG) | 27B-JANG | 권장 |
| **JANG 양자화** | +51% (9B) | 9B 이하 소형 | 소형 모델 권장 |
| SSD KV 캐시 | ~0% | — | 불필요 |
| vmlx | 불안정 | — | 미성숙 |
| mlx_lm prompt cache | Spec과 충돌 | — | 단독 사용 시만 |

### 3.2 Speculative Decoding 상세 (27B 기준)

27B + 0.8B 드래프트 모델의 25문제 풀셋 결과:

- **전체 평균**: +57~62% 속도 향상
- **최고 개선**: C4 (Lock-free HashMap) +208.7%
- **이상값**: M4 (수식) -55%, K4 (고전 한자어) -91%
- **이상값 원인**: 드래프트 모델이 수식/한자어 처리 실패 → 반복 재생성

**적용 규칙**:
```
✅ 효과 있는 경우:
- 동일 아키텍처 (동일 vocab)
- 드래프트:검증 크기비 > 10x
- Dense 모델
- 검증 모델 속도 ~14 tok/s 이하

❌ 역효과인 경우:
- MoE 모델
- JANG↔표준 혼용
- 이미 빠른 모델 (18+ tok/s)
- 수식·고전 한자어 집중 문제
```

### 3.3 27B 최적 조합 매트릭스

| 조합 | 응답시간 변화 | 판정 |
|------|-------------|------|
| 기본 (baseline) | — | 기준 |
| **+ Spec(0.8B)** | **-17~19%** | **최적** |
| + SSD-KV | +0.5~1.3% | 역효과 |
| + Spec + SSD | -7.6~10% | Spec 단독보다 나쁨 |
| + Spec + Prefix Cache | +2.2% | Spec 비활성화됨 (충돌) |

**결론: Spec(0.8B) 단독 적용이 최적**

## 5. 말씀 앱 프로젝트 적용 시사점

### 5.1 개발/테스트 환경 모델 추천

| 용도 | 추천 모델 | 설정 | 예상 성능 |
|------|---------|------|----------|
| **메인 개발** | Qwen3.5-27B + Spec(0.8B) | mlx_lm | ~27-30 tok/s |
| **최고 품질 검증** | Qwen3.5-122B | JANG 2S | ~22 tok/s |
| **빠른 반복** | Qwen3.5-9B-JANG | JANG 4S | ~27 tok/s |
| **수학/추론 검증** | R1-32B | mlx_lm | ~7 tok/s |

### 5.2 한국어 성능 관점

- **27B**: 한국어 **5/5** — 시조 분석, 높임법, 속담 모두 우수
- **122B**: 한국어 **5/5** — 전 영역 최고
- **R1 시리즈**: 한국어 **2~3/5** — 추론은 강하나 한국어 약점
- **JANG 9B**: K1에서 '진지(식사의 높임말)'를 혼동하는 오류

→ **말씀 앱의 한국어 성경 RAG에는 27B 또는 122B 권장**

### 5.3 RAG 파이프라인 연동 관점

| 항목 | 로컬 LLM 가능성 | 비고 |
|------|---------------|------|
| 개발 중 RAG 테스트 | 27B로 충분 | 한국어 품질 우수 |
| 임베딩 생성 | 별도 임베딩 모델 필요 | LLM과 별개 |
| 프로덕션 서빙 | Gemini 2.5 유지 | 로컬은 개발용 |
| 품질 벤치마크 | 122B로 최고 품질 기준선 | 프로덕션 품질 기준 |
| 오프라인 모드 (미래) | 9B 가능성 | 모바일 배포 시 검토 |

### 5.4 비용 절감 효과

```
개발 중 Gemini API 호출 → 로컬 27B 대체:
- Gemini 2.5 Pro: ~$3.5/M input + $10.5/M output
- 로컬 27B: $0 (전기료 제외)
- 일일 100회 테스트 기준 월 ~$50-100 절감 예상
```

## 6. 향후 벤치마크 계획

### 단기 (1~2개월)
- 122B 25문제 풀셋 재실험 (JANG 직접 런타임)
- 자동 채점 완성 → 효율 지수 계산
- 독립 세션 재실험 (공정 비교)

### 중기 (3~6개월)
- 차세대 모델 (Qwen4 등) 출시 시 재테스트
- 도메인별 드래프트 모델 탐색 (한국어 특화)
- Speculative Decoding 자동 온/오프 스위칭

### 장기 (6~12개월)
- MLX LoRA/QLoRA로 27B 도메인 적응 (성경/신학 특화)
- 모바일 온디바이스 추론 가능성 평가
- CI/CD LLM 품질 게이트 통합
