💡 검색 모드 선택 가이드

  • Cascading — 권 우선순위가 명확하고 비용 절감이 중요할 때 (예: 참부모님 직설 → 강해 → 해설 순). 지금 사용 중인 방식.
  • Weighted — 사람이 정한 가중치로 다중 소스를 합산. parallel_fusion으로 점진 이전 권장.
  • Parallel Fusion (신규) — A·B·C를 동시에 보고 RRF/Convex Combination으로 융합. Cascade의 "Tier 1에서 놓치면 끝" 문제 해결. "all" 봇과 멀티권 통합 검색에 최적.

🎯 핵심 메시지

  • 골격은 그대로, Stage만 추가/교체. 기존 11 Stage 파이프라인 구조를 깨지 않고 5.0·5.5·10번을 끼워넣고 8번 SearchStage에 옵션 하나 추가하는 방식.
  • 각 단계 도입은 평가셋 검증과 함께. 골든셋 50~100개 만들고 옵션 켜고 끄며 측정. 도메인·코퍼스에 따라 효과가 다르므로 무조건 적용 금지.
  • Phase 0가 가장 먼저. 자체 문서가 이미 적신호로 표시한 5개 설정 버그(score_threshold 0.60, REWRITE_TIMEOUT 0.8s 등) 수정만으로도 체감 검색 성공률이 크게 오를 가능성.
