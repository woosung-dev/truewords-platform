# 벡터DB 10종 종합 비교 분석

## 프로젝트 요구사항

- **데이터 규모:** 615권 x 200페이지 x 2~5청크 = 약 25만~60만 청크
- **언어:** 한국어 종교 텍스트 (고유 용어 다수)
- **필수 요건:** 하이브리드 검색, 메타데이터 필터링(다중 챗봇), 한국어 지원
- **운영 환경:** 셀프호스팅 가능, 소규모 팀(2명), 비용 효율 중요

## 개별 상세 분석

---

### 1. Qdrant ★★★★★ (8.75/10)

| 항목 | 내용 |
|------|------|
| **라이선스** | Apache 2.0 (완전 오픈소스) |
| **하이브리드 검색** | sparse + dense 벡터 네이티브, RRF 결합 |
| **최대 벡터 차원** | 65,535 |
| **메타데이터 필터링** | 매우 강력 (중첩 객체, 범위, 키워드, 지오 등) |
| **한국어** | 임베딩 모델 의존 + Mecab 기반 sparse 벡터로 BM25 구현 |
| **확장성** | 60만 청크 → 단일 노드 2~4GB RAM 충분 |
| **셀프호스팅** | Docker 1줄 배포, Snapshot API 백업 |
| **비용** | 셀프호스팅 ~$20-50/월, 클라우드 무료 티어 있음 |
| **커뮤니티** | GitHub ★22K+, Discord 15K+ |

**강점:** 가볍고, 하이브리드 검색 네이티브, payload 필터로 A|B 챗봇 조합 간편
**약점:** BM25를 위해 sparse 벡터를 직접 생성해야 함

---

### 2. Elasticsearch + nori ★★★★★ (8.25/10)

| 항목 | 내용 |
|------|------|
| **라이선스** | SSPL (ES) / Apache 2.0 (OpenSearch) |
| **하이브리드 검색** | 최고 수준 — BM25 + kNN + RRF 네이티브 |
| **최대 벡터 차원** | 4,096 ⚠️ (제한적) |
| **메타데이터 필터링** | 업계 최강 쿼리 DSL |
| **한국어** | nori 분석기 — 공식 한국어 형태소 분석, 사용자 사전, 동의어 |
| **확장성** | 수십억 문서 검증 |
| **셀프호스팅** | Docker, 패키지 |
| **비용** | 셀프호스팅 ~$50-100/월, Elastic Cloud ~$95/월~ |
| **커뮤니티** | GitHub ★75K+ (가장 성숙) |

**강점:** 한국어 BM25 압도적 (nori), 하이브리드 검색 최고
**약점:** 메모리 많이 필요, 벡터 차원 4,096 제한, 운영 복잡

### 3. Milvus / Zilliz ★★★★☆ (8.00/10)

| 항목 | 내용 |
|------|------|
| **라이선스** | Apache 2.0 |
| **하이브리드 검색** | v2.5부터 네이티브 BM25 + dense + sparse, RRF |
| **최대 벡터 차원** | 32,768 |
| **메타데이터 필터링** | SQL-like 표현식, JSON/배열 필드, 스칼라 인덱스 |
| **한국어** | Tantivy 기반 BM25에 Lindera 토크나이저 → 한국어 형태소 분석 |
| **확장성** | 수십억 벡터 설계 (분산 아키텍처) |
| **셀프호스팅** | Milvus Lite (단일 바이너리) ~ 풀 클러스터 |
| **비용** | Lite ~$20-30/월, 클러스터 ~$50-100/월 |
| **커뮤니티** | GitHub ★32K+ (가장 많은 스타) |

**강점:** 기능 풍부, 한국어 BM25 네이티브, 대규모 확장성
**약점:** 운영 복잡도 (etcd + MinIO), 60만에는 과한 인프라

### 4. Weaviate ★★★★☆ (7.65/10)

| 항목 | 내용 |
|------|------|
| **라이선스** | BSD-3 (오픈소스) |

| **한국어** | 기본 BM25 토크나이저가 공백/구두점 기반 → 한국어 형태소 약함 |

**강점:** BM25 내장으로 하이브리드 검색 가장 쉬움
**약점:** 한국어 BM25 토크나이저 부족 (교착어 미반영)
