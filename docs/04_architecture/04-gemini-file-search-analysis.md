# Gemini File Search API 분석

## 개요

2025년 11월 출시된 **완전 관리형 RAG 시스템**. 문서를 업로드하면 자동으로 청킹 → 임베딩 → 인덱싱 → 검색까지 처리.

## 작동 방식

```
문서 업로드 → File Search Store 생성
    ↓ (자동: 청킹, 임베딩, 인덱싱)
사용자 질문 → 벡터 유사도 검색 → 관련 청크 추출
    ↓
Gemini에 컨텍스트로 주입 → 답변 생성 + 출처 자동 인용
```

## 컨텍스트 윈도우

| 모델 | 입력 컨텍스트 | 출력 제한 |
|------|-------------|----------|
| Gemini 2.5 Pro | 1,048,576 토큰 (1M) | 65,536 토큰 |
| Gemini 2.5 Flash | 1,048,576 토큰 (1M) | 65,536 토큰 |

### 한국어 토큰 효율 (핵심 이슈)

| 언어 | 문자당 토큰 | 1M 토큰으로 커버 가능한 분량 |
|------|-----------|--------------------------|
| 영어 | ~0.25 토큰/문자 | ~1,500 페이지 |
| **한국어** | **~1.0-1.2 토큰/문자** | **~300-500 페이지** |

한국어는 영어 대비 **4-5배 비효율적**. SentencePiece 토크나이저가 영어 최적화 되어 있기 때문.

## 파일 업로드 제한

| 방법 | 최대 크기 | 비고 |
|------|----------|------|
| 인라인 | 100 MB | 요청 내 직접 포함 |
| Files API | 2 GB/파일 | 프로젝트 저장소: 20 GB, 48시간 후 만료 |
| Google Cloud Storage | 2 GB/객체 | GCS 연동 |

### File Search Store 용량

| 티어 | 총 프로젝트 저장 |
|------|----------------|
| Free | 1 GB |
| Tier 1 | 10 GB |
| Tier 2 | 100 GB |
| Tier 3 | 1 TB |

> 저장소는 입력 대비 ~3배 사용 (입력 + 임베딩). 스토어당 **20 GB 이하** 권장.

| 항목 | 내용 |
|------|------|
| 최소 토큰 | Flash: 1,024 / Pro: 4,096 |
| 기본 TTL | 1시간 (설정 가능) |

| 손익분기 | 시간당 4회+ 참조 시 비용 절감 |

## 615권 처리 가능성 분석

### 규모 추정

```
615권 x 200페이지 x 1,000자/페이지 = ~1.23억 한국어 문자
토큰 변환: ~1.23억 ~ 1.48억 토큰 (한국어 기준)
인덱싱 비용: ~$18.5 ~ $22.2 (1회성, 저렴)
저장: 무료
```

### 치명적 한계 (단독 사용 비추 이유)

| 한계 | 영향 |
|------|------|
| **메타데이터 필터링 없음** | A\|B 챗봇 조합 구현 **불가** |
| **커스텀 청킹 불가** | 권/장/절 계층 구조 반영 불가 |
| **하이브리드 검색 없음** | 순수 벡터만 → 종교 고유 용어 키워드 매칭 약함 |
| **커스텀 임베딩 모델 불가** | 한국어 최적화 임베딩 선택 불가 |
| **Re-ranking 불가** | 1차 검색 결과 정밀도 개선 불가 |
| **스토어당 20GB 권장** | 초과 시 레이턴시 저하 |
| **블랙박스** | 청킹/임베딩 품질 확인 불가 |

### 가장 큰 문제: 다중 챗봇 버전

```
❌ Gemini File Search:
   Store에 615권 전부 → 필터링 없이 전체에서만 검색
   → 말씀선집만 검색, 어머니말씀만 검색 등 불가

✅ Qdrant:
   payload: {book_type: "malssum", volume: 45, year: 1990}
   → filter: {book_type: ["malssum"]} → 말씀선집만 검색
   → filter: {book_type: ["malssum", "mother"]} → 조합 검색
```

## Google AI Studio vs Vertex AI

| 항목 | Google AI Studio (Gemini API) | Vertex AI RAG Engine |
|------|:---:|:---:|
| 대상 | 개발자, 스타트업, 프로토타입 | 엔터프라이즈, 프로덕션 |
| 커스터마이징 | 제한적 | 풀 컨트롤 (파싱, 청킹, 임베딩, 벡터 스토어) |
| 벡터 백엔드 | Google 관리 (불투명) | Vertex AI Search, Weaviate, Pinecone, 커스텀 |
| 보안 | API 키 | Google Cloud IAM, 감사 로그, VPC-SC |
| 스케일 | 1 TB (Tier 3) | 엔터프라이즈 스케일 (TB+) |

## 추천 활용 방식

**단독 사용 비추, 보조 역할로 활용:**

```
주 검색 엔진: Qdrant
  (하이브리드 검색, 메타데이터 필터, 다중 챗봇)
       ↓
생성 모델: Gemini 2.5 Flash
  + Context Caching (원리강론/대사전 ~200페이지 캐싱)
  → 캐시 적용 시 입력 비용 90% 절감
       ↓
File Search: 프로토타입/데모용으로만 활용
```

### Context Caching 비용 절감 시뮬레이션

```
항상 캐시할 문서 (원리강론 + 대사전 핵심 200페이지):
→ ~10만 토큰 (한국어 기준)
→ 캐시 저장: $0.10/시간
→ 캐시 입력: $0.03/1M 토큰 (Flash 기준, 90% 절감)

하루 1,000건 질문 시:
→ 캐시 없이: ~$30/일
→ 캐시 적용: ~$3/일 + 캐시 저장 $2.4/일 = ~$5.4/일
→ 월 ~$160 절감
```
