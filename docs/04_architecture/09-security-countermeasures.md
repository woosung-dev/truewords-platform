# 보안 및 악의적 사용 대응 방안

## 기본 6가지 방안

### 1. 인증 기반 접근 제어
- 회원 로그인 필수
- API 키 발급 제한

### 2. Rate Limiting
- IP/사용자별 분당 요청 수 제한

### 3. 답변 워터마킹
- "이 답변은 AI가 생성한 것이며..." 고지문 자동 삽입

### 4. 시스템 프롬프트 방어선 (준비 완료)
- 말씀 범위 이탈 질문 거부
- 정치/비방 유도 차단

### 5. 로그 모니터링
- 이상 패턴 탐지 (대량 크롤링, 악의적 질문 패턴)
- 자동 차단

### 6. 민감 인명/내용 필터
- 특정 인명/사건 언급 시 사전 정의된 가이드라인 답변 제공

## 추가 권장 사항

### 7. 단계적 공개 전략 (Staged Rollout)

```
Phase 1: 검색 기능만 제공
    → AI 해석 없이 원문만 보여줌 → 리스크 최소
    → 말씀 원문 검색 + 출처 표기

Phase 2: 내부 레드팀 대상 AI 답변 베타

Phase 3: 인증된 식구님 대상 제한 공개

Phase 4: 전체 공개
```

각 Phase에서 로그 분석 후 다음 단계로 진행.

### 8. 답변 범위 경계 (Guardrails)
- 말씀 **해석**이 아닌 말씀 **인용**에 집중하도록 시스템 프롬프트 설계
- "이 주제에 대해 아버님께서는 다음과 같이 말씀하셨습니다: [원문]" 형태
- AI가 독자적 해석을 덧붙이지 않도록 제한

### 9. 악의적 질문 패턴 DB 구축
- 레드팀 테스트 과정에서 발견된 악의적 질문 패턴을 DB화
- 유사 패턴 감지 시 자동 차단 또는 표준 답변 제공

## 종교적 언어 다층 해석 대응

### 문제
- 종교적 언어의 다층적 의미 해석 (예: "국가복귀" 용어)
- 같은 용어가 시대/맥락에 따라 다른 의미를 가질 수 있음

### 해결 방안
- 대사전 데이터를 **별도 용어사전 컬렉션**에 저장
- 시스템 프롬프트에 핵심 용어 100~200개의 정의 삽입
- 검색 시 관련 용어 정의를 동적으로 주입하여 LLM이 맥락에 맞게 해석
- 자세한 구조는 [06-terminology-dictionary-structure.md](./06-terminology-dictionary-structure.md) 참고
