# 프로젝트 개요 및 회의 정리

## 프로젝트 개요

차세대 플랫폼 개발의 일환으로 **말씀 AI 챗봇** 프로젝트를 시작.

- 기술적으로는 베타 버전 개발 완료 (Gemini File Search + pgvector FAQ) - 테스트 해당 부분 이어서 개발하지 않을것
- 핵심 과제: 신학적 학습과 리스크 관리
- 악의적 사용 방어 프롬프트 준비 완료

## 팀 구성

| 팀           | 역할        | 멤버                                        |
| ------------ | ----------- | ------------------------------------------- |
| **포너즈팀** | 기술 개발   | 김경서 대표, 장우성 개발자                  |
| **레드팀**   | 검증/테스트 | 이시카와, 김인지 교수, 박미성 등 + 생도분들 |

## 현재 개발 상태

- 사용자용 인터페이스: 여러 종류의 챗봇 선택 및 질문 가능
- 관리자용 페이지: 챗봇 생성, 문서 업로드(RAG), 시스템 프롬프트 설정, FAQ 관리
- 채팅 로그 데이터 수집 기능 구현 완료

## 학습 데이터 범위 (확정)

- 말씀선집, 어머니 말씀
- 원리강론, 대사전 등 검증된 자료 활용 예정

## 액션 아이템 현황

| 상태        | 항목                                              |
| ----------- | ------------------------------------------------- | ------- |
| ✅ 완료     | 학습 데이터 범위 결정 → 말씀선집, 어머니 말씀     |
| ✅ 완료     | 데이터 전달 방안 → 벡터DB 별도 운영 (Qdrant) 확정 |
| ✅ 완료     | 악의적 사용 방어 프롬프트 준비                    |
| 🔄 진행중   | 데이터 변환 (HWP→PDF/TXT) 및 파이프라인 구축      |
| 🔲 미정     | 레드팀 리스트 작성                                |
| 🔲 진행필요 | 대용량 아키텍처 개발                              |
| 🔲 진행필요 | 다중 챗봇 버전 (A, B, A                           | B 조합) |

## 속도의 중요성

- 외부에서 먼저 부모님 말씀 기반 AI를 만들 위험성 존재
- 일본에서 아버님 딥페이크 영상 제안 사례 있었음
- 완벽하지 않더라도 빠르게 출시하여 선점 필요

## 핵심 논의 사항

### 학습 데이터 범위와 기준

- 말씀 전집(615권), 어머니 말씀, 원리강론, 통일사상, 교수님 논문 등 포함 여부
- 최근 말씀 우선 vs 전체 말씀 균형 문제 → [05-rag-pipeline.md](./05-rag-pipeline.md) 참고
- 온라인 노출 시 악의적 사용 가능성 대응 → [09-security-countermeasures.md](./09-security-countermeasures.md) 참고

### 기술적 접근

- 파인튜닝보다는 RAG(문서 참조) 방식과 시스템 프롬프트가 더 효과적
- 3가지 조정 방법: 시스템 프롬프트(가장 강력), 참조 문서, FAQ
- 벡터라이징을 통한 맥락 기반 검색

### 향후 계획

1. 원리강론, 대사전, 최근 말씀을 기준으로 우선 학습 진행
2. 여러 버전의 챗봇을 만들어 비교 테스트
3. 검색 기능부터 먼저 제공하여 리스크 최소화 방안 검토
4. 레드팀 테스트를 통해 문제점 발견 및 개선
