# 앱 기능 정의서 (MVP)

## 1. 개요

시장 조사(doc 12), 경쟁 분석(doc 13), 성공 요인 전략(doc 14)을 종합하여 말씀 앱의 기능을 정의한다. MVP → Growth → Scale 3단계로 나누어 각 단계별 핵심 기능과 기술 스펙을 명시한다.

---

## 2. 기능 전체 맵

```
말씀 앱 기능 전체 구조
│
├── [Core] AI 말씀 대화
│   ├── 성경 Q&A (RAG 기반)
│   ├── 맞춤 기도문 생성
│   ├── 맥락 기반 성경 검색
│   └── 목양 상담 모드
│
├── [Daily] 일일 묵상 루틴
│   ├── 오늘의 말씀 카드
│   ├── 묵상 가이드 (텍스트/오디오)
│   ├── 감사 저널
│   └── 기도 트래커
│
├── [Retention] 습관 엔진
│   ├── 연속 묵상 Streak
│   ├── 영적 성장 시각화 (나무/정원)
│   ├── 성경 통독 진도표
│   └── 마일스톤 보상
│
├── [Audio] 오디오 콘텐츠 (Growth)
│   ├── 묵상 오디오 가이드
│   ├── 수면 성경 이야기
│   ├── 기도 명상 BGM
│   └── 백그라운드 재생
│
├── [Community] 커뮤니티 (Growth)
│   ├── 소그룹 함께 묵상
│   ├── 기도 제목 공유
│   ├── 그룹 챌린지
│   └── 익명 기도 부탁
│
├── [Bible] 성경 뷰어
│   ├── 역본 대조 읽기
│   ├── 하이라이트/북마크
│   ├── 상호참조 링크
│   └── 주석 보기
│
└── [B2B] 교회 솔루션 (Scale)
    ├── 교회별 맞춤 챗봇 (doc 07)
    ├── 교파별 데이터 라우팅 (doc 11)
    ├── 관리자 대시보드
    └── 셀/구역 관리
```

## 3. Phase별 기능 상세 스펙

### 3.1 MVP Phase (0~3개월)

**목표**: 핵심 가치 검증, 1,000 유저 확보

#### M-01: AI 말씀 대화 (핵심)

| 항목 | 스펙 |
|------|------|
| **기능** | 성경 기반 RAG Q&A + 맞춤 기도문 생성 |
| **AI 엔진** | Gemini 2.5 + Qdrant RAG (Phase 1 아키텍처) |
| **응답 방식** | SSE(Server-Sent Events) 스트리밍 |
| **출처 표시** | 모든 답변에 성경 구절 출처 명시 |
| **안전 장치** | 교리 생성 금지 + 유사도 임계값 + 톤 가이드라인 |
| **입력 검증** | Prompt Injection 방어 (doc 09) |
| **대화 저장** | 선택적 — 사용자가 저장/삭제 선택 |

**프론트엔드 고려사항**:
- SSE 기반 실시간 텍스트 렌더링 (타이핑 효과)
- 답변 내 성경 구절 탭 시 성경 뷰어 연동
- 대화 중 로딩 상태: Skeleton UI + "말씀을 찾고 있습니다..." 메시지
- 에러/타임아웃 시 우아한 Fallback UI + 재시도 버튼

#### M-02: 일일 묵상 카드

| 항목 | 스펙 |
|------|------|
| **기능** | 매일 새로운 말씀 카드 + 짧은 묵상 가이드 |
| **데이터** | 교회력/절기 반영 큐레이션 + AI 묵상 질문 |
| **갱신** | 자정 기준 새 콘텐츠 (타임존 고려) |
| **공유** | 이미지 카드 렌더링 → SNS 공유 |
| **캐싱** | 오프라인 Fallback (전일 캐시) |

**프론트엔드 고려사항**:
- 카드 형태 UI (스와이프 가능)
- html2canvas 또는 동등 라이브러리로 공유 이미지 생성
- React Query / SWR 캐싱 + 오프라인 모드

#### M-03: Streak 트래커

| 항목 | 스펙 |
|------|------|
| **기능** | 연속 묵상일 카운트 + 마일스톤 축하 |
| **마일스톤** | D+7, D+30, D+100, D+365 |
| **시각화** | 달력 히트맵 + 연속일 수 배지 |
| **알림** | 묵상 미완료 시 저녁 리마인더 (사용자 설정) |
| **복구** | 1회 면제권 (주 1회) — 완벽주의 부담 해소 |

**프론트엔드 고려사항**:
- Lottie/Rive 기반 축하 애니메이션
- 캘린더 히트맵 컴포넌트 (GitHub 스타일)
- 푸시 알림 네이티브 브릿지

#### M-04: 맞춤 기도문 생성

| 항목 | 스펙 |
|------|------|
| **기능** | 상황/감정 기반 개인화 기도문 AI 생성 |
| **입력** | 기도 카테고리 선택 + 자유 텍스트 |
| **카테고리** | 감사, 회개, 간구, 중보, 찬양, 치유, 지혜 |
| **톤** | 따뜻하고 위로적, 성경 구절 인용 포함 |
| **저장** | 기도문 저장 + 나중에 다시 기도하기 |

#### M-05: 성경 뷰어 (기본)

| 항목 | 스펙 |
|------|------|
| **기능** | 성경 읽기 + 하이라이트 + 북마크 |
| **역본** | 개역개정 (기본) + 추가 역본 확장 가능 |
| **렌더링** | Virtualization 기법 (대용량 텍스트 최적화) |
| **연동** | AI 대화에서 구절 탭 → 뷰어로 이동 |

### 3.2 Growth Phase (3~6개월)

**목표**: 리텐션 검증 + 차별화, 10,000 유저

#### G-01: 오디오 묵상 시스템

| 항목 | 스펙 |
|------|------|
| **콘텐츠** | 묵상 가이드 오디오, 수면 성경 이야기, 기도 BGM |
| **스트리밍** | HLS 기반 적응형 스트리밍 |
| **제어** | 재생/일시정지, 배속(0.5x~2x), BGM 볼륨 독립 조절 |
| **백그라운드** | OS 백그라운드 재생 + 잠금화면 제어 |
| **초기 소스** | TTS 생성 (비용 절감) → 이후 전문 녹음 |

**프론트엔드 고려사항**:
- 네이티브 오디오 모듈 브릿지 (React Native / Flutter)
- Mini Player (하단 고정) + Full Screen Player
- 60fps 프로그레스 바 애니메이션
- 오프라인 다운로드 기능

#### G-02: 소그룹 커뮤니티

| 항목 | 스펙 |
|------|------|
| **그룹 생성** | 초대 코드/링크 기반 소그룹 생성 |
| **함께 묵상** | 같은 말씀으로 소그룹원이 동시 묵상 + 나눔 |
| **기도 제목** | 기도 부탁 게시 + 기도 응답 축하 |
| **챌린지** | 그룹 40일 기도, 성경 통독 챌린지 |
| **프라이버시** | 익명 기도 부탁 옵션 |

**프론트엔드 고려사항**:
- Optimistic UI (즉각 피드백)
- 실시간 업데이트 (WebSocket 또는 Polling)
- 그룹 멤버 프로필 + 활동 피드

#### G-03: 시즌 챌린지 엔진

| 항목 | 스펙 |
|------|------|
| **교회력 연동** | 사순절, 대림절, 부활절, 추수감사절 |
| **챌린지 유형** | 40일 기도, 성경 통독, 감사 100가지 |
| **진도 시각화** | 전체/개인 진도율 + 그룹 랭킹 (비경쟁적) |
| **보상** | 완주 배지 + 특별 묵상 콘텐츠 |

#### G-04: 영적 성장 시각화

| 항목 | 스펙 |
|------|------|
| **메타포** | 나무/정원 성장 (초원의 새싹 확장 버전) |
| **성장 요소** | 묵상, 기도, 성경 읽기, 커뮤니티 참여 |
| **애니메이션** | Lottie/Rive 기반 벡터 애니메이션 |
| **공유** | 나의 정원 스크린샷 SNS 공유 |

---

### 3.3 Scale Phase (6~12개월)

**목표**: 수익화 + 교회 생태계, 100,000 유저

#### S-01: 교회 B2B 솔루션

| 항목 | 스펙 |
|------|------|
| **맞춤 챗봇** | 교회별 설교/강해 데이터 추가 학습 (doc 07) |
| **교파 라우팅** | 개혁/복음/오순절 등 교파별 답변 톤 조절 (doc 11) |
| **관리자 콘솔** | 사용 통계, 콘텐츠 관리, 멤버 관리 |
| **비용 모델** | 월 구독형 SaaS (교회 규모별 티어) |

#### S-02: 역본 대조 + 주석

| 항목 | 스펙 |
|------|------|
| **역본** | 개역개정, 새번역, NIV, ESV 등 |
| **주석** | 검수된 주석 데이터 연동 |
| **상호참조** | 구절 간 참조 링크 시각화 |
| **검색** | 의미 기반 성경 검색 (벡터 검색) |

#### S-03: 웰니스 확장

| 항목 | 스펙 |
|------|------|
| **수면 모드** | 수면 타이머 + 성경 이야기 + 기도 BGM |
| **감정 기반 추천** | 불안/감사/슬픔 등 감정 선택 → 맞춤 콘텐츠 |
| **주간 리포트** | 영적/정서적 건강 주간 요약 |

## 4. 기술 스택 요약

| 영역 | 기술 선택 | 근거 |
|------|---------|------|
| **모바일 앱** | **Flutter** (확정) | 크로스플랫폼, 네이티브 성능, Riverpod + go_router |
| **관리자 대시보드** | Next.js 16 (React) | 웹 기반 관리자 UI, Vercel 배포 |
| **AI 엔진** | Gemini 2.5 Pro | Phase 1 결정 유지 |
| **벡터 DB** | Qdrant | Phase 1 결정 유지 (8.75/10) |
| **운영 DB** | PostgreSQL | Phase 1 결정 유지 |
| **실시간 통신** | SSE (AI 답변) + WebSocket (커뮤니티) | 역할별 최적 프로토콜 |
| **오디오** | HLS + 네이티브 브릿지 | 배경재생 필수 |
| **캐싱** | Semantic Cache (doc 08) + React Query | 비용 절감 + UX 개선 |
| **애니메이션** | Lottie / Rive | 벡터 기반 경량 애니메이션 |
| **푸시 알림** | FCM (Android) + APNs (iOS) | 표준 |
| **이미지 생성** | html2canvas 또는 Skia | 공유 카드 렌더링 |

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|---------|
| **응답 시간** | AI 첫 토큰 < 2초, 성경 검색 < 500ms |
| **오프라인** | 일일 묵상 카드 + 성경 뷰어 오프라인 가능 |
| **접근성** | 큰 글씨 모드 (장년층), 다크 모드, VoiceOver/TalkBack |
| **보안** | 대화 내용 암호화, 무기억 세션 옵션 (doc 09) |
| **성능** | 60fps 애니메이션, 앱 시작 < 3초 |
| **데이터** | GDPR/PIPA 준수, 대화 데이터 사용자 삭제 가능 |
