# 데이터 소스 페이지 재설계: 카테고리 관리 + 업로드 개선

## 배경

데이터 소스 페이지에 3가지 문제:
1. 업로드 동작이 실제로 되는지 미확인 (백엔드 파이프라인 연결 검증 필요)
2. 카테고리 CRUD API는 완성되어 있으나 관리 UI가 없음 (DB 시드 4건만 존재)
3. 현재 페이지가 단일 기능(업로드)만 제공하여 관리 도구로서 부족

## 설계

### 1. 페이지 구조: 탭 기반

`/data-sources` 페이지를 탭 2개로 분리:

```
[문서 업로드]  [카테고리 관리]
─────────────────────────────
(선택된 탭 내용)
```

- **문서 업로드** (기본 탭): 기존 드래그앤드롭 + 대기 목록 + 처리 현황
- **카테고리 관리**: 카테고리 목록 테이블 + 추가/수정 다이얼로그

탭 상태는 클라이언트 `useState`로 관리 (URL 반영 없음).
통계 카드 4개는 탭 위에 항상 표시.

### 2. 카테고리 관리 탭

#### 테이블 (목록)

| 컬럼 | 내용 |
|------|------|
| Key | 읽기 전용 배지 (A, B, C, D 등) |
| 이름 | 말씀선집, 어머니말씀 등 |
| 설명 | 615권 텍스트 데이터 등 |
| 색상 | 색상 원형 칩 |
| 검색 가능 | 체크/언체크 아이콘 |
| 활성 | 활성/비활성 배지 |
| 액션 | 편집 / 비활성화 버튼 |

#### 추가/수정 다이얼로그

Sheet 컴포넌트(이미 설치됨)를 사용하여 사이드 패널로 열기:

- **key**: 추가 시 텍스트 입력 (영문+숫자+언더스코어, 생성 후 변경 불가). 수정 시 읽기 전용
- **이름**: 텍스트 입력 (필수)
- **설명**: 텍스트 입력
- **색상**: 7개 프리셋 중 선택 (indigo, violet, blue, slate, emerald, amber, rose). 색상 원형 버튼
- **검색 가능**: 체크박스 (false면 검색 티어에서 제외)
- **활성**: 수정 시에만 토글 가능

#### 비활성화

"비활성화" 클릭 시 확인 다이얼로그 → soft delete (is_active=false).
이미 Qdrant에 적재된 데이터는 영향 없음 (source 필드는 유지).

### 3. 업로드 동작 확인

- 로컬 백엔드 실행 + 파일 업로드 테스트
- Qdrant 미연결 시 에러 메시지가 제대로 표시되는지 확인
- 업로드 버튼 클릭 → 백엔드 202 응답 → 처리 현황 폴링 동작 확인

### 4. 사용 컴포넌트

- **탭**: 버튼 기반 (shadcn Tabs 없으므로 직접 구현)
- **테이블**: 기존 shadcn Table 컴포넌트
- **추가/수정 폼**: shadcn Sheet (사이드 패널)
- **색상 선택**: 커스텀 색상 피커 (7개 원형 버튼)
- **확인 다이얼로그**: window.confirm 또는 인라인 확인

## 검증

1. `npx tsc --noEmit` — TypeScript 오류 없음
2. 브라우저: 탭 전환 동작 확인
3. 브라우저: 카테고리 추가/수정/비활성화 확인
4. 브라우저: 파일 업로드 → 처리 현황 갱신 확인
